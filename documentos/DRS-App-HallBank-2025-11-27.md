# Informações Gerais  

**Nome do projeto:** App Bancário – “HallBank”  
**Versão do documento:** 1.0  
**Data:** 27/11/2025  
**Autor:** Felipe Barbosa  

**Descrição:**  
Este documento especifica escopo, restrições, requisitos funcionais, não funcionais e requisitos de interface  
para o protótipo do aplicativo HallBank. O objetivo é definir claramente o comportamento, as características e os limites do sistema.  

## Escopo

O protótipo consiste em:  

Tela de Login  
Tela Principal (Dashboard)  
Tela Extra 1: Transferência PIX  
Tela Extra 2: Extrato Bancário  

## Restrições

1. O app será entregue somente como protótipo, sem backend real.  
2. A interação será limitada ao que o Figma permitir.  
3. Somente funcionalidades definidas no escopo serão implementadas.  

## Requisitos Funcionais (RF)

**RF01 - Autenticação**  
O protótipo deve permitir que o usuário faça login utilizando CPF e senha.  
**RF02 - Exibir Saldo**  
O protótipo deve exibir saldo no dashboard.  
**RF03 - Transferências**  
O protótipo deve apresentar funções de transferência por PIX, TED e pagamento de boletos.  
**RF04 - Ações Rapidas**  
O protótipo deve apresentar no dashboard botões para ações rápidas de transferência.  
**RF05 - Investimentos**  
O protótipo deve exibir uma área de investimentos.  
**RF06 - Pesquisa**  
O protótipo deve apresentar um campo de pesquisa.  
**RF07 - Cartões**  
O protótipo deve exibir uma área para acessar os cartões.  
**RF08 - Perfil**  
O protótipo deve exibir uma área para acesso as informações do perfil do usuário.  

## Requisitos não Funcionais (RNF)

**RNF01 – Usabilidade**  
O layout deve seguir padrões modernos e apresentar navegação simples e intuitiva.  
**RNF02 – Consistência Visual**  
Todas as telas devem usar a mesma paleta, espaçamentos e tipografia.
